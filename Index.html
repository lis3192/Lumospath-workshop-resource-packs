The Mental Models

Internalise these six ideas first and every other feature clicks. Start here.

Mental Model #1

It's not Chat vs Claude — it's the right tool for the right job. ChatGPT's still better for image generation. Claude wins for thinking, writing, and running operations. You don't need to migrate everything. Migrate what fits.

Mental Model #2

Audit your workflow before you automate. AI is moving fast. The trap is automating things that don't actually move the needle. Look at what's broken in your business right now, fix the bottleneck, then bring AI in. Don't get sucked into the hype.

Mental Model #3

Connectors are utensils. Skills are the recipe. You need both. Connectors plug Claude into your tools. Skills tell Claude how to use them. Without skills, Claude has the equipment but no instructions on what to cook.

Mental Model #4

Treat Claude like a junior employee. A skill is an SOP — tone of voice, branding rules, step-by-step process. Update it as you spot gaps, the same way you'd update an SOP for a new hire. Don't expect it to read your mind. Brief it.

Mental Model #5 — Models & Tokens

Opus — strategy and coding. Burns roughly 30% more tokens.
Sonnet — implementation. Where you do most of your work.
Haiku — quick lookups. The "where should I eat in Tokyo" tier.
Picking the right model is the cheapest way to extend your token budget.

Mental Model #6 — Start Messy, Then Systematise

Don't try to build the perfect workflow on day one. Use Claude in regular chat first. Mess around, ask things, get answers. When you notice yourself repeating the same request — that's your signal to turn it into a skill. The system builds itself from the bottom up. Not the top down.

💡 The junior staff member rule

Day one with a new hire, you don't hand them the keys to everything. Same with Claude. Start each connector on "Ask me" before you do something. Loosen permissions as trust builds. That's how you stay safe without being scared of the tools.

Connectors

This is where Claude stops being just a chatbot and starts working with your actual tools.

What connectors actually are

Also called MCPs (Model Context Protocol — Anthropic built the standard). Think of them as USB-C for AI. Instead of API keys and config files, you log in once with your normal account and the connection is made. Claude can now talk to that tool — read, draft, organise — based on the permissions you give it.

How to add a connector

→ Settings → Customise → Connectors → "+ Browse Connectors" → click your tool → log in

Common connectors for service businesses

📁

📅

Daily briefings, meeting prep, schedule scanning. Pairs well with Gmail.

📝

🎙️

Granular permissions per connector

Most people don't realise this — every connector gives you tool-by-tool permissions. You can let Claude search Gmail but block reading full threads. Or let it draft but never send. Or set every action to "Ask me" so Claude checks with you each time.

→ Settings → Customise → Connectors → click any connector → toggle each permission

💡 Start tight, loosen later

For anything that writes, sends, or deletes — start on "Ask me" until you trust the workflow. That's the junior staff rule again. Give it more access as you learn what you're comfortable with.

Custom MCPs (when there's no official connector)

Not every tool has an official connector yet. If yours is missing, search Google for "[tool name] MCP". Most major tools (Blotato, n8n, WordPress, Shopify) have a public MCP setup page where you copy the URL and paste it into Claude.

⚠️ Be careful with GitHub MCPs

Community-built connectors live on GitHub. Some have malware or sketchy permissions baked in. Before you connect any unofficial MCP: paste the URL into Claude Cowork and ask it to review the source code for anything suspicious before I install. Stick to official tools or ones with strong star counts and recent maintenance.

A skill is an SOP for Claude — a markdown file that teaches it exactly how to do a specific task your way, every time. Like writing detailed instructions for a junior employee — except this employee follows them perfectly, forever.

How to create a skill

→ make sure Skill Creator is ON → in chat, type "/skill-creator" → describe what the skill should do → answer Claude's questions → review → "Copy to skills"

💡 The "I love this output" rule

Any time you do something in Claude and love how it came out — immediately turn it into a skill. Just say: "Use Skill Creator to turn what we just did into a repeatable skill." Claude has the whole conversation thread in front of it, knows exactly what good looks like, and writes the skill from that.

The description field is everything

When you're chatting with Claude, it reads each skill's description to decide which one to use. Vague description = wrong skill (or no skill). Be specific: what does it do, and when should Claude reach for it?

⚠️ Don't have every skill turned on at once

Too many active skills = too much context for Claude to sort through. Toggle skills on/off based on what you're working on. Settings → Customise → Skills → toggle off the ones you don't need right now.

Artifacts & Projects

Artifacts are the real files Claude creates. Projects are your persistent business workspaces.

What artifacts are

An Artifact is a real, usable file Claude creates inside the chat — not just a block of text. It lives in a panel to the right of the conversation. You can preview it, publish it, download it, or copy the code into your own site.

📊

Interactive Dashboards

HTML pages with charts, filters, clickable elements. Like the resource pack you're reading right now.

📄

Documents

Word docs, PDFs, formatted reports. Ready to send to clients.

📈

Spreadsheets

Excel files with formulas, formatting, and data. Downloadable.

💻

Code

Python scripts, HTML pages, JavaScript. Runnable, copy-pasteable.

Publishing an artifact

→ Artifact panel → "Publish Artifact" button → copy the shareable link

💡 Works like a Google Doc with "anyone with the link" access

Want to pull it offline? Hit Unpublish and the link dies. The code behind it is just HTML / CSS / JS — you can also copy it and host it on your own website or on Vercel.

Projects

A Project is its own dedicated workspace inside Claude — with its own memory, files, and instructions that carry across every chat inside it. Think of it like a client folder with the brief tucked inside — except Claude reads the brief automatically, every single time you open it.

→ Left sidebar → New Project → name it → Add instructions (paste your system prompt) → Add content (upload docs or connect a Google Drive file) → start chatting
Why Projects matter

Persistent memory — Claude remembers context across every chat inside the project.
Dedicated files — upload documents or connect a Google Drive folder.
Scheduled tasks — automation tied to that specific project.
Use one Project per client, per offer, per launch. Stop mixing contexts in one chat.

💡 The Google Drive sync advantage

Connect a Google Drive file to your Project and update it later — Claude's version updates automatically. ChatGPT can't do this. If your processes change, update the doc and Claude catches up next time you open the Project.

⚠️ Sharing projects = Team plan only

If you want to share a Project with your team, you need a Team plan. Individual accounts can't share Projects with others. You can still publish artifacts from inside a Project though — those work on any plan.

FAQ

An Artifact is a real file Claude creates inside the chat — HTML page, dashboard, document, code. You can publish it as a shareable link. A Project document is a file you upload (or connect via Google Drive) for Claude to reference within that workspace. Different jobs.

Cowork

Claude's autonomous desktop agent. You delegate a task, Claude works through it — you don't have to guide every step.

The shift from Chat to Cowork

Regular Claude chat = conversation. You go back and forth.
Cowork = delegation. You hand Claude a task, it grabs your files, connects to your tools, runs your skills, and delivers the finished output.

⚠️ Desktop app only — and your computer needs to stay on

Cowork only runs on the Claude Desktop app (Mac and Windows 11+). Your computer has to be on and the app open for scheduled tasks to fire. You don't have to be sitting at it — but it can't run with the laptop closed or asleep.

Chat vs Cowork vs Code — when to use which

Mode Best for Limitations
Chat

Cowork
(Desktop app only)		 Claude does the work — creates files, organises folders, runs skills, schedules recurring tasks. The "delegation" mode. Desktop app only. Your laptop has to stay on for scheduled tasks. Token-heavy.
Code
(Desktop app only)		 Full developer terminal. Custom integrations, advanced automations, agent building. Overwhelming for non-developers. We're not covering it in this pack.

When Cowork isn't enough

If a workflow needs to run reliably in the cloud — when your laptop's closed or asleep — Cowork can't help. That's where third-party automation tools come in: n8n, Zapier, or Make. They run on their own servers, cost cents per workflow, and don't depend on your computer being awake. Use them for mission-critical automations and keep Cowork for enrichment.

Tokens & Models

The single fastest way to stop hitting your usage limits is picking the right model — not upgrading your plan.

What "tokens" actually means

Tokens are how Claude measures usage — bits of text it reads and writes. Every conversation, file upload, and skill run costs tokens. The more context Claude has to scan through, the more it burns. Hit your limit and your session resets after a few hours.

Match the model to the task

Model Best for Token use Think of it as...
Opus Strategy, planning, complex coding, deep reasoning. ~30% more than Sonnet — burns fast. The senior strategist.
Sonnet Implementation. Writing, editing, drafting, building. Your everyday model. Standard. Cost-effective for most work. The doer.
Haiku Quick lookups, simple questions, "what's a good restaurant in Liverpool?" Lowest cost. Fast and cheap. Glorified Google.

💡 The handoff workflow

Use Opus to build the strategy or plan. Then open a new chat, switch to Sonnet, and have it implement the plan. You get the senior thinking without burning Opus tokens on the execution work. Single biggest token saver.

Skills save tokens too

When you ask Claude to do something without a skill, it scans your full chat history, custom instructions, and context to figure out what you mean. That's a lot of tokens. With a skill, Claude reads the small description, picks the right one, and only loads the full workflow when needed. Repeat-tasks-as-skills = your second biggest token saver.

Don't waste tokens here

⚠️ Skip the Design feature for now

Claude's built-in Design tool is clunky and burns tokens for outputs that aren't great. Use Canva, Gemini, or ChatGPT for image generation instead. Don't get sucked into the hype — the tool will get better, but right now it's a token drain.

Anthropic Skills on GitHub

Official skills published by Anthropic — browse, review, or download as a starting point. Best practice: drop the URL into Claude Cowork and ask it to review the skill before you install (protects against prompt injection).

🔍

Finding custom MCPs

If a tool isn't in the official connector list, Google "[tool name] MCP" — most major tools (Blotato, n8n, WordPress, Shopify) maintain a public MCP setup page. Avoid unofficial ones with low star counts on GitHub. Prefer official or high-star maintained options.

☁️

n8n, Zapier, Make — for cloud automation

Use these when a workflow has to run reliably in the cloud and your laptop's not always on. n8n is self-hosted (~$20–30 USD/month, fully featured). Zapier and Make are SaaS alternatives — easier to start, less powerful at scale.
